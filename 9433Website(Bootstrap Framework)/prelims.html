World Wide Web

History of the World Wide Web. Difference of the World Wide Web and the internet. Hypertext Transfer Protocol, the protocol used by the world wide web

History

The World Wide Web is invented primarily on information sharing between scientists in the year of 1989 by Sir Tim Berners Lee and considered as the father of the World Wide Web .The World Wide Web's three fundamental technologies, the HyperText Markup Language(http), the Uniform Resource Identifier (URI) and the Hypertext Transfer Protocol.The WWW is a large web resource where people can access through internet and is identified by a URL(Uniform Resource Locator) e.g http://www.microsoft.com.

World Wide Web vs Internet

Internet is the interconnection of computers globally that makes them communicate. In the internet, information are able to travel through protocols - rule or the agreed format that is used to transmit data.

Hypertext Transfer Protocol

Hypertext Transfer Protocol is the protocol used by the World Wide Web

Description

Hypertext transfer protocol is an application protocol that is being used by the World Wide Web. The protocol defines how messages are formatted and transmitted and what are the proper actions of Web servers and browsers should do with regards to the commands. HTTP runs on top of TCP/IP using RP port 80 by default or TCP port 80 by default or TCP port 443 for HTTPS(HTTP over SSC/RS) and is based on a client server architecture. HTTP uses a request-response standard protocol - client(browsers, robots and search engines) sends an HTTP request message to the server and the server(Web server) processes the request and replies with an HTTP response message. It is also considered as stateless communications protocol because servers dont keep information about clients in between requests. HTTP provides support for other functionalities such as: cache control, content media type(MIME) specification, language and character set specification, content/transfer codings, content negotiation and client server protocol negotiations.

HTTP Addressing

The Uniform Resource Locator(URL) also known as web address is used to locate a computer network to retrieve web resources. Web addresses are typically on the address bar of a web browser located at the top. URL is a type of URI (Uniform Resource Identifier).
HTTP Address Syntax:
http : // hostname [ : port ] / path [ ? searchwords ]
Scheme: http or the protocol to be used to access the resource location.
hostname: the internet hostname (e.g www.microsoft.com)
port: the port number part
path: the path within the host

Client Server Communication

Clients sends request to the server in the form of a request method. And then The server is the one who will accept and will respond to the client. It responds with a status line, including the message's protocol version and a success or error code.

HTTP Message Header Fields

General-header field is applicable for both request and response messages.
Request-header field is applicable only for request messages.
Response-header field is applicable only for response messages.

  • Accept Ranges: allows partial e.g Accept ranges: bytes
  • Age: used for cache control.
  • Etag: size of resource and time stamp
  • Location: redirection
  • Proxy Authenticate
  • Retry-After: returns service error
  • Server: info about the server(does not require header)
  • Vary: indicates value
  • WWW Authenticate

Entity-header field defines meta-information about the entity-body or about the resource identified by the request.

  • Allow: methods can call from entity content encoding
  • Content-Language: accessibility perspective
  • Content Length
  • Content-Location
  • Content MD5: hashing algorithm (deprecated header)
  • Content-Range: range request
  • Content Type: e.g (text/plain, application/javascript)
  • Expires: controlling cache access
  • Last-Modified

HTTP MESSAGE ("Client")

The HTTP Message is composed of a Request Line,Message Headers and a Message Body. The request contains client information.


Request Line

The Request-Line begins with a method token, followed by the Request-URI and the protocol version, and ending with CRLF. The elements are separated by SP characters. No CR or LF is allowed except in the final CRLF sequence.

Message Headers

This could be a General,Request,and/or an entity Header

Message Body

The Message body is optional on a Request message, it is also called the payload

Example of an http Request message

GET /hello.htm HTTP/1.1
User-Agent: Mozilla/4.0 (compatible; MSIE5.01; Windows NT)
Host: www.tutorialspoint.com
Accept-Language: en-us
Accept-Encoding: gzip, deflate
Connection: Keep-Alive

HTTP RESPONSE MESSAGE ("Server")

The HTTP Message is composed of a Status Line,Message Headers, empty line and a Message Body.


Status Line

The Status line contains the HTTP protocol version, status code and a reason phrase

Message Headers

This could be a General,Response,and/or an entity Header

Message Body

The Message body is optional on a Request message, it is also called the payload

Example of an http Request message

HTTP/1.1 200 OK
Date: Mon, 27 Jul 2009 12:28:53 GMT
Server: Apache/2.2.14 (Win32)
Last-Modified: Wed, 22 Jul 2009 19:15:56 GMT
Content-Length: 88
Content-Type: text/html
Connection: Closed
GET /hello.htm HTTP/1.1

HTTP Methods

GET, HEAD, POST, PUT, DELETE,OPTIONS, TRACE and Connect.

GET- it retrieves data and should not affect the data when retrieving it

HEAD- transfers the status line and the header section

POST- it is used to submit data to the server

PUT- replaces the representations of the target resource with the uploaded content

DELETE- removes the representations of the target resource given by URI

CONNECT- establish a tunnel to the server, Allows the client and the server to exchange credentials

OPTIONS- describe the communication options for the target resource

TRACE- performs a message loopback
